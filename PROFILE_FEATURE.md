# 👤 Функция профиля пользователя

## Описание

Добавлена **опциональная** система профилей с расчетом нормы калорий по формуле Миффлина-Сан Жеора. Пользователи могут использовать бота без профиля или заполнить профиль для получения персональной рекомендации по суточной норме калорий.

## ⚠️ Важно: Профиль не обязателен!

Бот полностью функционален без профиля:
- ✅ Анализ фото еды работает без профиля
- ✅ Подсчет калорий и БЖУ работает без профиля
- ✅ Все основные функции доступны без профиля

Профиль нужен только для:
- 📊 Расчета персональной нормы калорий
- 🎯 Отслеживания прогресса по цели
- 📈 Показа процента от дневной нормы при анализе блюд

## Возможности

### 1. Создание профиля
- Пол (мужской/женский)
- Возраст (в годах)
- Вес (в килограммах)
- Рост (в сантиметрах)
- Уровень физической активности

### 2. Расчет нормы калорий
Используется формула Миффлина-Сан Жеора:

**Для мужчин:**
```
BMR = (10 × вес в кг) + (6.25 × рост в см) − (5 × возраст) + 5
```

**Для женщин:**
```
BMR = (10 × вес в кг) + (6.25 × рост в см) − (5 × возраст) − 161
```

**Суточная норма:**
```
Калории = BMR × Коэффициент активности
```

### 3. Уровни физической активности

| Уровень | Коэффициент | Описание |
|---------|-------------|----------|
| Малоподвижный | 1.2 | Сидячая работа, минимум движения |
| Легкая активность | 1.375 | Прогулки, зарядка 1-3 раза в неделю |
| Умеренная активность | 1.55 | Тренировки 3-5 раз в неделю |
| Высокая активность | 1.725 | Интенсивные тренировки 6-7 раз в неделю |
| Очень высокая | 1.9 | Интенсивные тренировки 2 раза в день |

### 4. Управление профилем

Команда `/profile` открывает меню с опциями:
- **Заполнить заново** - создать новый профиль или обновить существующий
- **Изменить цель** - изменить норму калорий
  - Ввести вручную (500-10000 ккал)
  - Пересчитать по формуле
- **Удалить профиль** - удалить все данные профиля

## Структура базы данных

### Таблица `user_profiles`

```sql
CREATE TABLE user_profiles (
  telegram_id BIGINT PRIMARY KEY,
  gender VARCHAR(10),           -- 'male' или 'female'
  age INTEGER,                  -- возраст в годах
  weight DECIMAL(5, 2),         -- вес в кг
  height INTEGER,               -- рост в см
  activity_level DECIMAL(3, 3), -- коэффициент активности (1.2-1.9)
  calorie_goal INTEGER,         -- целевая норма калорий
  is_manual_goal BOOLEAN,       -- установлена вручную или рассчитана
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);
```

## API

### ProfileService

```javascript
const profileService = require('./services/profileService');

// Получить профиль
const profile = await profileService.getProfile(telegramId);

// Создать/обновить профиль
const result = await profileService.createOrUpdateProfile(telegramId, {
  gender: 'male',
  age: 30,
  weight: 75,
  height: 180,
  activityLevel: 1.55
});

// Обновить цель вручную
await profileService.updateCalorieGoalManually(telegramId, 2500);

// Пересчитать цель
await profileService.recalculateCalorieGoal(telegramId);

// Удалить профиль
await profileService.deleteProfile(telegramId);
```

### CalorieCalculator

```javascript
const calorieCalculator = require('./services/calorieCalculator');

// Рассчитать калории
const { bmr, dailyCalories } = calorieCalculator.calculate({
  gender: 'male',
  weight: 75,
  height: 180,
  age: 30,
  activityLevel: 1.55
});

// Получить уровни активности
const levels = calorieCalculator.getActivityLevels();
```

## Валидация

### Ограничения
- Вес: 1-500 кг
- Рост: 1-300 см
- Возраст: 1-150 лет
- Коэффициент активности: 1.2-1.9
- Цель калорий (ручная): 1-10000 ккал

### Проверки
- Все поля обязательны при создании профиля
- Числовые значения должны быть положительными
- Пол должен быть 'male' или 'female'
- Коэффициент активности должен быть из предустановленного списка

## Безопасность

- Данные профиля привязаны к `telegram_id` пользователя
- При удалении пользователя профиль удаляется автоматически (CASCADE)
- Все входные данные валидируются перед сохранением
- SQL-инъекции предотвращены использованием параметризованных запросов
